HTML

  • You can think of HTML as the structure or foundation of a house. HTML is the first step to building a great webpage!
  • The head element contains information (or metadata) about the webpage.
  • Metadata is information about data. It's stored in the <head> and is used by the Browser, Search Engines and other developers.
  • The body element represents the visible content shown to the user.
  • Inside the body you add other HTML elements. You should aim to use semantic HTML elements wherever possible.
  • Semantic HTML elements describe their role on the page, making the overall webpage more accessible.

CSS

  • Returning to the analogy of building a house, CSS can be thought of as the design/decor of the house.
  • You can add CSS to a webpage in 3 ways: in-line CSS; internal stylesheet; external stylesheet. An external stylesheet is preferred.
  • You declare CSS rules via a (selector { property: value; }) combo.
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.

JavaScript

  • Last but not least is JavaScript which adds behaviour/functionality to a webpage. Using the house anaology from earlier, think of it as making the doors open and blinds open/close. Most webpages would be super boring if not for JavaScript!
  • A variable is a container/binding that stores items. Using the name of a variable you can easily refer to the data stored inside.
  • JavaScript makes the webpage come to life through interactivity and dynamic behaviour!

Git

  • Git is a version control system with a purpose to make software development easier.
  • Version control helps to save snapshots or versions of a project, allowing a project to easily be restored to a past point if needed.
  • Other collaborators can create snapshots of their own to work on the project alongside you, but working on their own and other developers.
  • git checkout -b branch-name: creates a new branch and switches to it.
